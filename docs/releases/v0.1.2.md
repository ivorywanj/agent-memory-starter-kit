# Agent Memory Starter Kit v0.1.2

Agent Memory Starter Kit v0.1.2 improves the first-run experience for real users.

## What Shipped

- Agent-led first-run wizard in `docs/first-run-wizard.md`.
- README quickstart now treats `answers.example.json` as a demo fixture, not the real-user default path.
- Project initialization can record workspace pointers with `Project name | workspace folder`.
- Workspace pointers are saved for later task routing but are not read, scanned, indexed, or imported during setup.
- Public release check now verifies the first-run wizard and workspace pointer documentation.

## Quickstart

```bash
git clone https://github.com/ivorywanj/agent-memory-starter-kit.git
cd agent-memory-starter-kit
```

Copy the prompt from `docs/first-run-wizard.md` into Codex, Claude Code, Cursor, or another local-file-reading Agent.

After the Agent asks one question at a time, it can initialize the runtime:

```bash
scripts/memory --root ./my-agent-memory init \
  --name "Alex" \
  --language "English" \
  --work-type "indie product builder" \
  --communication-style "conclusion first, plain language" \
  --project "Example SaaS | ~/projects/example-saas"
```

## Validation

```bash
python3 tests/test_public_package.py
python3 scripts/memory_guard.py
python3 scripts/public_release_check.py
```

## Safety Boundary

The initializer stores project workspace paths only as routing pointers. It does not read workspace files or import folder contents. Secret-like project or workspace values are blocked by the guard.

## Known Limits

- CLI + Markdown only.
- Single-user local runtime.
- First-run wizard is prompt/documentation driven; there is no web UI.
- Workspace existence is recorded as an init-time status only.
