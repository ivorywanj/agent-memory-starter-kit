# Agent Memory Starter Kit v0.1.3

Agent Memory Starter Kit v0.1.3 adds shared-runtime bridge files for multiple Agents.

## What Shipped

- New `memory share` command.
- Codex bridge target: `AGENTS.md`.
- Claude Code bridge target: `CLAUDE.md`.
- Cursor bridge target: `.cursor/rules/agent-memory.mdc`.
- Generic Agent bridge target: `AGENT_MEMORY.md`.
- Public docs for sharing one runtime across Agent workspaces.
- Tests that verify bridge files are pointer-only and do not copy user profile or project memory.

## Quickstart

```bash
scripts/memory --root ./my-agent-memory share --agent codex --workspace ./my-project
scripts/memory --root ./my-agent-memory share --agent claude --workspace ./my-project
scripts/memory --root ./my-agent-memory share --agent cursor --workspace ./my-project
```

Use `--append` when the target file already exists.

## Validation

```bash
python3 tests/test_public_package.py
python3 scripts/memory_guard.py
python3 scripts/public_release_check.py
```

## Safety Boundary

Agent-specific bridge files are pointers only. They must not copy user profile, project facts, hot memory, history, session cache, deprecated audit, secrets, or customer data.

## Known Limits

- CLI + Markdown only.
- Bridge generation does not install editor extensions or change external Agent settings.
- Existing target files are not overwritten unless `--force` or `--append` is explicit.
