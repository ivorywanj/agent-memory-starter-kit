# JourneyMem v0.1.1

JourneyMem is a local-first memory runtime for coding agents. It gives agents a compact Markdown startup path, a short-term session cache, safety checks, and an automatic `remember -> recall -> improve -> forget` loop.

## What Shipped

- Product-focused README for external users.
- Checked-in release notes for GitHub Release publishing.
- Stronger public release check that verifies required README sections and release notes.
- No runtime command changes from v0.1.0.

## Quickstart

```bash
curl -fsSL https://raw.githubusercontent.com/ivorywanj/agent-memory-starter-kit/main/install.sh | bash
```

Then run `memory` and choose `memory new` or `memory connect`. The user does not hand-write Markdown.

## Validation

```bash
python3 tests/test_public_package.py
python3 scripts/memory_guard.py
python3 scripts/public_release_check.py
```

GitHub Actions runs the same public checks on push and pull request.

## Safety Boundary

Do not store or print API keys, tokens, webhook URLs, database URLs, private keys, raw sessions, customer data, account data, cookies, or passwords. The public release check also blocks private markers and secret-like content in public package files.

## Known Limits

- CLI + Markdown only.
- Single-user local runtime.
- No hosted backend, auth layer, graph database, vector database, or web UI.
- Benchmark evidence is published only as an anonymized summary; raw runs and private runtime paths are excluded.
