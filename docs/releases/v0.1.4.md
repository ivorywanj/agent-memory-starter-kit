# Agent Memory Starter Kit - Productized Shortcuts

This release makes the starter kit easier for external users to operate through three simple entries: new setup, Agent connection, and backup.

## What Shipped

- New user-facing shortcut: `/memory new`.
- New user-facing shortcut: `/memory connect`.
- New user-facing shortcut: `/memory backup`.
- README first screen now explains the three-choice flow in lower-terminology language.
- New `docs/productized-user-flow.md` with measurable T21-T26 acceptance criteria.
- New workflow docs for `memory-new`, `memory-connect`, and `memory-backup`.
- Public fixture tests now verify menu behavior, Agent connection, backup zip exclusions, and first-screen terminology.

## Quickstart

```text
/memory
/memory new
/memory connect
/memory backup
```

The Agent runs the matching command:

```bash
scripts/memory new
scripts/memory connect
scripts/memory backup
```

## Validation

```bash
python3 tests/test_public_package.py
python3 scripts/memory_guard.py
python3 scripts/public_release_check.py
git diff --check
```

## Safety Boundary

Backups exclude `.env*`, temporary dialogue data, search indexes, raw runs, generated drafts, and local database files. Connection files are pointer-only and must not copy user profile, project facts, hot memory, history, observed memory, audit records, or secrets into Agent workspaces.

## Known Limits

- CLI + Markdown only.
- Single-user local library only.
- No hosted sync or cloud backup in this release.
- The slash-command wording is implemented as Agent-facing workflow documentation plus CLI commands; editor-native slash command installation is not included.
